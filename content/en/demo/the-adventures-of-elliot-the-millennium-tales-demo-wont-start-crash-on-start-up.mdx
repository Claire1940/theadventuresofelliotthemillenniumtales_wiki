If you have managed to get past the initial crash but the game is highly unstable or closes shortly after reaching the main menu, you may need to adjust your display configurations outside of the game. 

Official documentation notes that the game supports V-sync on (up to 120 fps) and V-sync off (selectable 30, 60, or 120 fps). However, allowing the frame rate to spike uncontrollably during loading screens can trigger GPU safety shutdowns, resulting in a crash to the desktop.

| Setting | Recommended Value | Why It Helps Prevent Crashes |
|---|---|---|
| Max Frame Rate | 60 FPS | Prevents GPU overheating during uncapped menu screens |
| V-Sync | Enabled (On) | Stabilizes frame pacing and synchronizes with monitor refresh rate |
| Display Mode | Borderless Windowed | Prevents DX12 exclusive fullscreen alt-tab crashes |
| Hardware Accelerated GPU Scheduling | Enabled (Windows Settings) | Improves resource allocation for the HD-2D rendering pipeline |

To apply a hard frame rate cap without accessing the in-game menu (which you cannot reach if the game is crashing), open the **NVIDIA Control Panel** or **AMD Software: Adrenalin Edition**. Navigate to your 3D application settings, manually add the game's executable, and set the Max Frame Rate limit to 60 FPS. 

Additionally, if you are using a multi-monitor setup with different refresh rates, temporarily disable your secondary monitors using `Windows Key + P` and selecting "PC screen only." Some DX12 applications struggle to initialize when polling multiple displays with conflicting Hz values.
